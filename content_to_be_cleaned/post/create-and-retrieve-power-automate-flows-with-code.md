As you are probably aware, Microsoft is looking to [eliminate SharePoint
2010 and 2013
workflows](https://support.microsoft.com/en-us/office/sharepoint-2010-workflow-retirement-1ca3fff8-9985-410a-85aa-8120f626965f?ui=en-US&rs=en-US&ad=US){.markup--anchor
.markup--p-anchor}. In fact, 2010 workflows have already been retired,
while 2013\'s retirement date(s) are still looming. 

 

[Microsoft has advised users to manually rebuild their necessary
SharePoint workflows (SPWF's) as comparable
]{style="font-family: inherit;"}[M365 Power
Automate](https://docs.microsoft.com/en-us/power-automate/getting-started){.markup--anchor
.markup--p-anchor}[ flows. To that end, Microsoft has provided some
]{style="font-family: inherit;"}[basic
guidance](https://docs.microsoft.com/en-us/sharepoint/dev/business-apps/power-automate/guidance/migrate-from-classic-workflows-to-power-automate-flows){.markup--anchor
.markup--p-anchor}[, including ]{style="font-family: inherit;"}[a
tool](https://docs.microsoft.com/en-us/sharepoint/dev/transform/modernize-scanner){.markup--anchor
.markup--p-anchor}[ to sniff out SPWF's within your cloud
environment. ]{style="font-family: inherit;"}

When it comes to planning the migration of your SPWF's to Power Automate
flows, it helps to think of them as two completely different platforms.
Despite that fact that it's all just 'Microsoft' under the covers, Power
Automate is a substantially more robust, complex, and powerful
application than SharePoint Designer. It is simply not 'apples to
apples', which is why you won't find a convenient 'CONVERT' button. 

 

That said, there are opportunities for automating the creation of Power
Automate flows. This will potentially go a long way towards helping
organizations who have invested heavily in SPWF's and now have a
mountain of work ahead of them to convert. In this post, I will walk
through the details of programmatically creating Power Automate flows
with links to a [sample C# console
app](https://github.com/TracyGH/CreatePowerAutomateFlow/tree/master/CreateFlows).

###   {#toc-hId--1502704976 .graf .graf--h3}

### Part 2 --- Get ClientData Content for New Flows {#toc-hId-984807857 .graf .graf--h3}

In order to create flows programatically, we will use the [Power
Automate Management Web
API](https://docs.microsoft.com/en-us/power-automate/web-api){.markup--anchor
.markup--p-anchor}. The meat and potatoes of the flow payload is
contained in the `clientdata`{.markup--code .markup--p-code}
[property](https://docs.microsoft.com/en-us/power-automate/web-api#create-a-cloud-flow){.markup--anchor
.markup--p-anchor}. To help determine appropriate content for populating
this section, I recommend manually creating a 'base' Power Automate flow
using the UI. Then, you can [leverage the
API](https://docs.microsoft.com/en-us/power-automate/web-api#list-flows){.markup--anchor
.markup--p-anchor} to get the `clientdata`{.markup--code
.markup--p-code} content from that manual flow and copy/paste it into
the payload for new flow(s).

1.  Follow the instructions for creating a new Solutions flow as shown
    above in Part 1. 
2.  Navigate to your tenant [Power Platform admin
    center](https://admin.powerplatform.microsoft.com/environments){.markup--anchor
    .markup--li-anchor} to obtain the environment URL. Update the
    variable at line #17 with the relevant URL. 
3.  Uncomment the `RetrieveFlow`{.markup--code .markup--li-code}method
    call at [line
    #40](https://github.com/TracyGH/CreatePowerAutomateFlow/blob/master/CreateFlows/SampleProgram.cs)
    and confirm the`CreateFlow`{.markup--code .markup--li-code} method
    at [line
    #41](https://github.com/TracyGH/CreatePowerAutomateFlow/blob/master/CreateFlows/SampleProgram.cs)
    is commented out.
4.  See Part 1 above to obtain a Flow ID (GUID) for the flow you just
    created. Update the variable at [line
    #113](https://github.com/TracyGH/CreatePowerAutomateFlow/blob/master/CreateFlows/SampleProgram.cs)
    with the flow GUID value. 
5.  Insert a breakpoint at [line
    #122](https://github.com/TracyGH/CreatePowerAutomateFlow/blob/master/CreateFlows/SampleProgram.cs)
6.  Run the project, providing credentials for a user who has access to
    the [Dataverse Web
    API](https://docs.microsoft.com/en-us/powerapps/developer/data-platform/authenticate-oauth){.markup--anchor
    .markup--li-anchor}. (I have only tested with a global admin
    account.)
7.  Copy the string value within the `clientdata`{.markup--code
    .markup--li-code} .

### Part 3 --- Programmatically Create a New Flow {#toc-hId--822646606 .graf .graf--h3}

1.  Paste the `clientdata`{.markup--code .markup--li-code} content from
    above into the variable located on line #61.
2.  Comment out the `RetrieveFlow`{.markup--code .markup--li-code}method
    call at [line
    #40](https://github.com/TracyGH/CreatePowerAutomateFlow/blob/master/CreateFlows/SampleProgram.cs)
    and confirm the`CreateFlow`{.markup--code .markup--li-code} method
    at [line
    #41](https://github.com/TracyGH/CreatePowerAutomateFlow/blob/master/CreateFlows/SampleProgram.cs)
    is *not* commented out.
3.  Run the project!

If everything works as expected, you should be able to locate your new
flow by following the instructions in Part 1 to view Solutions
flows. Test your new flow to confirm everything works as expected. 

 

### Part 4 --- 'Tweak' the ClientData Content {#toc-hId-1664866227 .graf .graf--h3}

Below is a de-serialized example of the content within the
`clientdata`{.markup--code .markup--p-code}property of a Power Automate
flow. This particular flow is [triggered when a new
item](https://docs.microsoft.com/en-us/sharepoint/dev/business-apps/power-automate/sharepoint-connector-actions-triggers#when-an-item-is-created){.markup--anchor
.markup--p-anchor} is created within a SharePoint list. The indicated
`table`{.markup--code .markup--p-code}property at line #52 holds the
value for the GUID of the relevant SharePoint list.

 

To create a new Power Automate flow which targets a different SharePoint
list (within the same Site), simply update that GUID value in the
`clientdata`{.markup--code .markup--p-code} . While I have only tested
this specific scenario, the potential applications for 'tweaking' values
to facilitate bulk flow creation are endless.
